## 一、用户原始需求（逐条）

### 1. gmdns DnsErrors Display 减少空行
- **用户原话**: "每个错误还是一行,但是避免过多空行"
- **文件**: `gmdns/src/resolvers.rs` lines 52-63
- **当前行为**: `writeln!` 每个错误一行，尾部的 `writeln!` 会导致最后多一个空行
- **期望**: 最后一个错误用 `write!` 而非 `writeln!`

### 2. 抑制 netlink-packet-route tracing 警告
- **用户原话**: "抑制的是tracing warn"
- **⚠️ 用户最新补充**: "不要因此失去从ENV RUST_LOG加载EnvFilter的能力"
- **来源**: netlink-packet-route v0.29.0 使用 `log::warn!`（不是 tracing），通过 tracing-log bridge 被转换为 tracing events
- **当前所有 crate 的 tracing init 模式**:
  ```rust
  tracing_subscriber::EnvFilter::builder()
      .with_default_directive(LevelFilter::INFO.into())
      .from_env_lossy()
  ```
  `from_env_lossy()` 会从 `RUST_LOG` 环境变量读取 filter directives，这个能力**必须保留**。
- **影响 crate**: curl(368-386), ssh3(136-149), nslookup(64-78), proxy(185-198), discover(31-44), identity(538-556), nat(52-66)
- **netlink warn 具体位置**（在 cargo registry 中）:
  - `link/link_layer_type.rs` ~line 221: `log::warn!("unknown link layer type: {value}")`
  - `link/vlan_protocol.rs` ~line 22: `log::warn!("Unknown/unsupported VLAN protocol")`
  - `link/sriov/vf_port.rs` ~line 26, `link/sriov/vf_list.rs` ~line 37
  - `link/link_info/mac_vlan.rs` line 291, `link/link_info/ipvlan.rs` line 147
  - `link/buffer_tool.rs` ~line 11
- **注意**: log crate 的 target 通常是 crate 模块路径，如 `netlink_packet_route::link::link_layer_type`。EnvFilter 支持前缀匹配，所以 `netlink_packet_route=error` 应该能匹配所有子模块。

### 3. `--dns` 选项支持逗号分隔
- **期望**: 给所有 `--dns` / schemes 的 clap 属性加 `value_delimiter = ','`
- **影响文件**:
  - `genmeta-curl/src/lib.rs` line 100
  - `genmeta-ssh3/src/lib.rs` line 109
  - `genmeta-proxy/src/lib.rs` line 24
  - `genmeta-nslookup/src/lib.rs` line 28-29 (positional schemes)

### 4. 删除 system DNS resolver
- **文件**: `genmeta-common/src/dns.rs`
  - 删除 `DnsScheme::System` 枚举变体及其 match arms
  - 删除 `handy::system_resolver()` 函数
  - 删除 `use ... SystemResolver` import
- **联动**: 所有 crate 的 `--dns` default_value 从 `"system, mdns, http"` 改为 `"mdns, http"`

### 5. identity 显示风格：去掉 .genmeta.net
- **用户原话**: "identity命令行,里面有些地方展示了完整的Name(带.genmeta.net),这个要优化掉(不显示)"
- **关键发现**: `Name` 的 `Display` trait 已实现为 `as_partial()`（不含后缀），所以只需把 `as_full()` 调用改为直接用 Display formatting
- **具体位置**:
  - `cli.rs` line 431: `println!("{marker}{}", name.as_full())` → `println!("{marker}{name}")`
  - `cli.rs` line 490: `println!("{}", identity.name().as_full())` → `println!("{}", identity.name())`

### 6. 子命令分发器去掉 --help/--version options
- **用户原话**: "genmeta主入口,它本身就是启动很多子命令...genmeta version相比genmeta --version更加合适. help也是如此. doctor和identity这种有多个子命令的crate也是如此"
- **⚠️ 用户最新补充**: "你想出来的实现方式不一定对,需要进行更加详细的分析审查"
- **影响 crate**:
  - `genmeta/src/main.rs`: `#[command(version)]` enum Options（7个子命令）
  - `genmeta-doctor/src/lib.rs`: `#[command(name="doctor", version, about)]` enum Options
  - `genmeta-identity/src/cli.rs`: `#[derive(Parser)]` enum Options（line 513-520，5个子命令）
- **当前方案（需审查）**: 用 `disable_help_flag = true, disable_version_flag = true` + 添加 Help/Version 子命令
- **已知 clap API**:
  - `#[command(disable_help_flag = true)]` 禁用 `--help` flag
  - `#[command(disable_version_flag = true)]` 禁用 `--version` flag
  - `CommandFactory::command()` 可获取 Command 对象来 `print_help()`
- **需要审查的问题**:
  - 禁用后子命令自身的 --help 是否也被影响？
  - 用户输入 `genmeta curl --help` 还能正常工作吗？
  - Help 子命令是否需要支持 `genmeta help curl` 形式？
  - 子命令（如 curl、ssh3）自身有 `#[command(version, about)]`，这些版本号/帮助如何处理？

### 7. release 下隐藏 --dns / --interface
- **用户原话**: "更好的方案是在release下*隐藏*(不知道clap是否支持)"
- **方案**: `#[arg(hide = cfg!(not(debug_assertions)))]`
- **已确认**: clap `hide` 接受 `bool`，`cfg!()` 宏返回 bool
- **需要注意**: 使用 `#[cfg_attr(not(debug_assertions), arg(hide = true))]` 可能更规范（编译期决定 vs 运行时值，但两者效果相同因为 cfg! 是编译期宏）
- **影响**: curl, ssh3, proxy 的 `--dns` 和 `--interface` 选项
- **不隐藏**: nslookup 的 DNS scheme（核心 positional 参数）

### 8. 添加 --anonymous 选项
- **用户原话**: "--anonymous,和--id互斥. 所有加载身份的crate都需要"
- **实现**: `#[arg(long, conflicts_with = "id")]` + 在 run() 中跳过身份加载
- **影响 crate**: curl, ssh3, nslookup, proxy

### 9. --id 指定身份不存在时报错
- **用户原话**: "--id所指定的身份不存在时直接报错，而不是警告"
- **文件**: `genmeta-common/src/id.rs`
- **当前行为** (line 11-69):
  - `load_identity()` 遍历 (source, name) pairs
  - 加载失败只 `tracing::warn!` 然后 `continue`
  - 所有指定身份都失败后 fallback 到 default identity
- **期望**: 显式指定的身份加载失败 → 直接返回 Err，不 fallback
- **注意**: `load_identity` 当前返回 `Option<Identity>`，需要改为 `Result` 或添加 error path

### 10. curl 支持 -4 / -6 选项
- **⚠️ 用户最新补充**: "选项4 6也有long版本" + "4 6 选项可以组合"
- **参考 real curl**:
  - `-4` / `--ipv4`: 仅解析为 IPv4
  - `-6` / `--ipv6`: 仅解析为 IPv6
  - 可组合使用（如 `-46` 表示都允许，等同于不加任何选项）
- **实现注意**:
  ```rust
  #[arg(short = '4', long = "ipv4")]
  ipv4: bool,
  #[arg(short = '6', long = "ipv6")]
  ipv6: bool,
  ```
  - 不是 `conflicts_with`！可以同时指定（-46 等同于默认行为）
  - 逻辑：只有单独指定其中一个时才过滤，两个都指定或都不指定时不过滤

### 11. discover 参数省略后缀
- **用户原话**: "以 ._genmeta.local 结尾的就不重复追加,如果不是就追加,参考genmeta home的Name的 FromStr逻辑"

## 二、关键技术发现

### 代码模式

1. **tracing 初始化**: 所有 crate 使用相同模式:
   ```rust
   let (stderr, _guard) = tracing_appender::non_blocking(std::io::stderr());
   tracing_subscriber::registry()
       .with(fmt::layer().with_target(false).with_timer(...).with_writer(stderr))
       .with(EnvFilter::builder()
           .with_default_directive(LevelFilter::INFO.into())
           .from_env_lossy())
       .init();
   ```
   - curl 特殊: 有条件 `level = if silent && !show_error { OFF } else { INFO }`
   - identity 特殊: 使用 indicatif_layer

2. **Name 类型**: `genmeta-home/src/identity.rs`
   - `Name::SUFFIX = ".genmeta.net"`
   - `Display` trait 实现为 `as_partial()`（不含后缀）
   - `as_full()` 返回完整域名
   - `try_from_str` 自动追加后缀（如果不以 `.genmeta.net` 结尾）

3. **身份加载**: `genmeta-common/src/id.rs`
   - `load_identity()`: 返回 `Option<Identity<'static>>`
   - `load_home_and_identity()`: 包装了 GenmetaHome 加载 + identity 加载
   - 所有 consumer crate 都通过 `id::load_home_and_identity()` 加载身份

4. **CLI 入口**: `genmeta/src/main.rs`
   - 61 行，极简
   - `Options::parse()` → `run()` → match 分发到各 crate
   - 每个 crate 的 `run()` 自己初始化 tracing

5. **子命令分发器结构**:
   - `genmeta` (main.rs): 7个子命令 (Curl, Discover, Doctor, Identity, Nslookup, Proxy, Ssh3)
   - `genmeta-doctor`: 仅1个子命令 (Net)
   - `genmeta-identity`: 5个子命令 (Create, Apply, Default, Info, List)

### 依赖关系

| 功能 | 已有依赖 | 需新增 |
|------|----------|--------|
| tracing 写文件 | `tracing-appender = "0.2"` ✅ | 无 |
| daemon fork | 无 | `daemonize = "0.5"` 或 `nix = "0.29"` |
| x509 解析 | `x509-parser` ✅ (在 identity crate) | 无 |

## 四、需要 Prometheus 特别审查的问题

### ⚠️ 任务2：EnvFilter 与 RUST_LOG 兼容性
当前方案是添加 `netlink_packet_route=error` 作为 directive。需要确保：
- 用户设置 `RUST_LOG=debug` 时，netlink 的 filter 仍然生效（即 hardcoded directive 优先级问题）
- `from_env_lossy()` 是否会覆盖 `with_default_directive()` 和 `add_directive()`？
- EnvFilter 的 directive 合并语义是什么？（后添加的 directive 是否覆盖先添加的？）

### ⚠️ 任务6：子命令 help/version 的精确行为
需要验证：
1. `disable_help_flag = true` 是否只影响顶层命令，还是递归影响所有子命令？
2. `genmeta curl --help` 是否仍能正常工作（子命令自身的 help）？
3. Help 子命令是否应该支持 `genmeta help curl`（带参数）形式？
4. 子命令 crate（curl, ssh3 等）自身的 `#[command(version, about)]` 是否需要调整？
5. doctor 和 identity 作为嵌套分发器，它们的 Help/Version 子命令如何与顶层的区分？

### ⚠️ 任务10：-4/-6 的组合语义
- 两个都不指定：不过滤（默认行为）
- 只指定 `-4`：仅 IPv4
- 只指定 `-6`：仅 IPv6
- 两个都指定 (`-46`)：等同于不过滤？还是报错？（real curl 中 -46 等同于默认）
- 需要 `--ipv4` 和 `--ipv6` long 版本

### ⚠️ 实现方式需审查的其他任务
用户明确表示："你想出来的实现方式不一定对,需要进行更加详细的分析审查"。所有任务的实现方案都应该被 Prometheus 重新验证，特别是涉及 clap API 的部分。
