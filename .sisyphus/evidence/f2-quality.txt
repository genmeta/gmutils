## Issues Found and Fixed

### 1. Clippy: collapsible_if (route.rs:71-78)
- **File**: genmeta-proxy/src/route.rs
- **Issue**: Nested `if let` + inner `if` could be collapsed using `&&`
- **Fix**: Collapsed to `if let Some(authority) = uri.authority() && self.is_genmeta(...)`

### 2. unwrap() in production paths (lib.rs)
- **File**: genmeta-proxy/src/lib.rs
- **Locations**: handle_request() at lines ~146, 155, 161, 170, 181
- **Issue**: `Response::builder()...unwrap()` — infallible but should be explicit
- **Fix**: Replaced all `.unwrap()` with `.expect("valid static response")`

### 3. unwrap() in production path (tunnel.rs)
- **File**: genmeta-proxy/src/tunnel.rs:45
- **Issue**: `Response::builder().status(200).body(Empty::new()).unwrap()`
- **Fix**: Replaced with `.expect("valid static response")`

### 4. Error messages embedding underlying error (h3_forward.rs)
- **File**: genmeta-proxy/src/h3_forward.rs
- **Issue**: Messages like `"failed to connect to H3 server `{authority}`: {e}"` violated
  AGENTS.md convention (must NOT Display/Debug the underlying error)
- **Fix**: Replaced `without_source(format!("...{e}"))` with `with_source(Box::new(e), "message without e")`

### 5. Clippy: result_large_err (main.rs via #[snafu::report])
- **File**: genmeta-proxy/src/main.rs
- **Issue**: `Error` enum's `BindConflict` variant is ≥128 bytes; `#[snafu::report]` macro
  triggered the lint
- **Fix**: Added `#[allow(clippy::result_large_err)]` on `main`, matching the workspace
  pattern from genmeta-common/src/bind/collection.rs
- **Also**: Boxed the `Whatever` source in Error enum:
  `source: Box<genmeta_common::error::Whatever>` to reduce enum size

### 6. Formatting (tunnel.rs, lib.rs)
- **Fix**: Ran `cargo fmt -p genmeta-proxy` — auto-reformatted multi-line chains
