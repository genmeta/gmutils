#### 1. gmdns DnsErrors Display 减少空行
- **文件**: `gmdns/src/resolvers.rs` (lines 52-63)
- **当前**: `writeln!` 每个错误一行，尾部多空行
- **改为**: 每个错误仍一行，但避免过多空行。将最后一个 `writeln!` 改为 `write!`，或去掉多余换行
- **方案**: 
  ```rust
  // Before:
  writeln!(f, "All DNS resolvers failed")?;
  for (resolver, error) in &self.errors {
      writeln!(f, "`{resolver}` failed: {}", Report::from_error(error))?;
  }
  // After: 去掉尾部多余换行
  writeln!(f, "All DNS resolvers failed")?;
  for (i, (resolver, error)) in self.errors.iter().enumerate() {
      if i < self.errors.len() - 1 {
          writeln!(f, "`{resolver}` failed: {}", Report::from_error(error))?;
      } else {
          write!(f, "`{resolver}` failed: {}", Report::from_error(error))?;
      }
  }
  ```
- **状态**: 待实现

### 第二批：CLI 结构改造

#### 6. 子命令分发器去掉 --help/--version options
- **目标**: `genmeta version` 代替 `genmeta --version`；`genmeta help` 代替 `genmeta --help`
- **影响 crate**:
  - `genmeta/src/main.rs` — 主入口 Options enum
  - `genmeta-doctor/src/lib.rs` — Options enum
  - `genmeta-identity/src/cli.rs` — Options enum (line 513)
- **方案**: clap 的 `#[command(disable_help_flag = true, disable_version_flag = true)]` + 添加 `Help` 和 `Version` 子命令变体
  ```rust
  #[derive(Parser)]
  #[command(disable_help_flag = true, disable_version_flag = true)]
  enum Options {
      // ... existing subcommands ...
      /// Show help information
      Help,
      /// Show version information  
      Version,
  }
  ```
- **状态**: 待实现

#### 7. `cfg(debug_assertions)` 隐藏 --dns / --interface
- **目标**: release 下隐藏这些选项（仍可使用，只是 --help 不显示）
- **方案**: clap 支持 `#[arg(hide = true)]`，用 cfg 控制：
  ```rust
  #[arg(long, value_name = "scheme", default_value = "mdns, http", 
        hide = cfg!(not(debug_assertions)))]
  ```
  注意: `hide` 接受 `bool`，`cfg!()` 宏返回 bool — 完美匹配
- **影响**: curl, ssh3, proxy 的 `--dns` 和 `--interface` 选项
- **nslookup**: DNS scheme 是核心 positional 参数，不隐藏
- **状态**: 待实现

#### 8. 添加 --anonymous 选项
- **flag**: `--anonymous`
- **行为**: 跳过身份加载，强制 `without_identity()`
- **与 --id 互斥**: `#[arg(long, conflicts_with = "id")]`
- **影响 crate**: curl, ssh3, nslookup, proxy（所有加载身份的 crate）
- **实现**:
  ```rust
  /// Use anonymous mode (no client identity)
  #[arg(long, conflicts_with = "id")]
  anonymous: bool,
  ```
  在 run() 中：
  ```rust
  let id = if options.anonymous {
      None
  } else {
      id::load_home_and_identity(...).await?
  };
  ```
- **状态**: 待实现

#### 9. --id 指定身份不存在时报错
- **文件**: `genmeta-common/src/id.rs`
- **当前**: `load_identity()` 对指定身份加载失败只 warn，继续尝试 default
- **改为**: 如果有显式指定的身份且加载失败 → 直接返回 Err，不 fallback 到 default
- **方案**: 修改 `load_identity()` 签名，增加参数或改变行为：
  - 当 `tried_specified == true` 且所有指定身份失败 → return Err 而非 warn + fallback
  - 需要将 `load_identity` 返回类型从 `Option<Identity>` 改为 `Result<Option<Identity>, Error>`
- **状态**: 待实现

#### 10. curl 支持 -4 / -6 选项
- **文件**: `genmeta-curl/src/lib.rs`

  ```rust

  #[arg(short = '6')]
  ipv6: bool,
  ```

  ```rust
  bind::setup_bind_interfaces_with(binds, |bind_uris| {

      if options.ipv4 {
          bind_uris.retain(|uri| uri.family() != Some(Family::V6));
  ```

#### 18. proxy --daemon (fork+setsid) 和 --log
- **文件**: `genmeta-proxy/src/lib.rs`
- **依赖**: 需要 `nix` crate（当前不在 workspace 中）
- **--daemon 实现**:
  - 使用 `nix::unistd::fork()` + `nix::unistd::setsid()`
  - 父进程 fork 后退出，子进程继续运行
  - 子进程 setsid 创建新 session
- **--log 实现**:
  - 接受文件路径参数
  - 子进程负责打开文件（所以 `/dev/stdout` 是子进程的 stdout）
  - 将 tracing subscriber 的 writer 指向该文件
  ```rust
  /// Run as a daemon (fork to background)
  #[arg(long)]
  daemon: bool,
  
  /// Log output destination (file path, opened by child process)
  #[arg(long, requires = "daemon")]
  log: Option<PathBuf>,
  ```
- **状态**: 待实现
